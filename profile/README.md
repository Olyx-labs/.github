# Olyx

Control plane for AI and MCP-native execution.

Olyx helps teams route, trace, and govern model + tool calls across providers without hardcoding provider-specific logic into application code.



## Why Teams Use Olyx

- **Provider-agnostic routing:** central execution policy across multi-provider AI stacks  
- **Traceable execution:** per-request routing context, tool calls, cost, and latency signals  
- **MCP-aware governance:** scoped tool execution with project-level controls  
- **Fast adoption path:** OpenAI-compatible gateway first, full SDK integration next  

## Core Capabilities

- Project-scoped execution isolation  
- Routing and policy controls  
- End-to-end trace visibility  
- Cost and latency observability  
- Governance-ready operational workflows

We’re building developer-first infrastructure for reliable, governable, and cost-aware AI execution at scale.
